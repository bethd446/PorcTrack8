Notes design · Mockup 5 · Final

Closer · décisions, hésitations, effort

5 décisions tranchées

Patterns retenus

  • Auth signup : long scroll 1 page (pas wizard 3 étapes). Yao remplit 5 champs en 90 s, pas de tunnel multi-écrans à abandonner.
  • Photo crop : 4:3 forcé (mémoire device + sujet animal). Carré 1:1 trop serré pour une loge, ratio libre = chaos galerie.
  • Pull-to-refresh : custom V76 (Loader2 mono + label uppercase). Le natif Ionic est trop blanc/neutre pour le DNA terrain.
  • PWA install : sticky top sur Today (pas floating bottom). Le bottom est déjà saturé par BottomNav + toasts undo + sheets.
  • Push permission : trigger métier 1ʳᵉ saillie enregistrée (pas onboarding C.5). Yao comprend pourquoi quand le contexte est concret — pas en abstrait au step 5.
2 hésitations détaillées

À débattre avec dev

  • Email confirmation H.4 : faut-il une polling auto côté front (toutes les 4 s vérifier si le user a cliqué) ou laisser le user revenir manuellement ? Polling = meilleure UX mais coûte 1 requête/4 s × N users en attente. Compromis : polling 30 s puis fallback manuel.
  • Swipe-to-delete sur alertes vs truies : même geste, conséquences très différentes. Sur truie = soft-delete + undo 5 s OK. Sur alerte = "marquer traité" = changement métier réel. Risque de confusion. Option B : sur alerte, swipe = action contextuelle (acquitter), pas destructif.
Top 3 priorités d'implémentation

À porter d'abord

  • Auth flow complet · Supabase Auth déjà branché — UI uniquement, 2 j. Bloquant pour tout le reste.
  • Photo upload Capacitor · Camera + Crop natifs, autorisations, tags. 4 j. Débloque 5 fiches qui ont déjà des slots photo.
  • Push permission custom prompt · trigger 1ʳᵉ saillie + ré-prompt 7 j. 1,5 j. Débloque la valeur métier #1 (ne pas rater une MB).
