Notes design · décisions · effort

3 décisions tranchées · 2 hésitations · récap série v76

3 décisions design fortes

01 · Marius

Bulles colorées · pas typo pure

Bulle utilisateur fond --pt-accent opaque cream, bulle Marius fond --pt-bg ink. Typo pure (alignement seul) testée mais perd la lisibilité sur message long. Markdown render minimal Marius : titres mono, listes à puces ambre, code inline ambre. Évite le mimétisme ChatGPT — style cahier d'éleveur.

02 · Long press

Action sheet bottom · pas inline expansion

Inline expansion testée — casse le rythme de la liste et masque la ligne longue après. Sheet bottom avec header eyebrow + entité + 4-5 actions verticales = pattern unifié avec quick-add modals. Touch targets 48px, dernière action Voir fiche sert de fallback navigation.

03 · DataTable

Cards stackées mobile · table desktop

Pas de horizontal scroll forcé sur mobile : illisible avec 9 colonnes. Cards stackées avec eyebrow + KV grid 2 cols = densité mobile correcte. Desktop ≥1024px reprend la table HTML traditionnelle avec sort + export CSV. Toggle "Mode avancé" dans Réglages, pas par défaut.

2 zones d'hésitation

Hésité · Toast

Stack 3 max ou LIFO 1 visible ?

Stack vertical 3 max permettrait plus de feedback en saisie rapide (pesée groupée 50 porcelets). Choisi : LIFO 1 visible + auto-dismiss 4s + queue silencieuse. Yao en pesée bat la cadence ≥1 toast/seconde — 3 stack se chevauchent visuellement et masquent le BottomNav. La queue laisse passer le dernier toast pertinent ; le précédent disparaît avant que l'œil ait le temps de le manquer.

Hésité · Empty porcelets

CTA "Voir bandes en gestation" ou message neutre ?

CTA contextuel aurait été utile pour Aïssa qui débugge. Choisi : message neutre sans CTA. Les porcelets sont une conséquence métier (apparaissent après MB), pas un objet à créer manuellement. Le CTA "Ajouter porcelet" existe en quick-add D.4 pour les rares cas d'import — il n'a pas vocation à être promu en empty state.
