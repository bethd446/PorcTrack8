Règles d'usage

5 do · 5 don't · snippet officiel

✓ 1 primary par écran

Une seule action métier dominante. Si 2 actions semblent égales, choisir laquelle promouvoir et passer l'autre en secondary ou ghost.

✓ Hauteur ≥ 44 px sur mobile

Cibles tactiles primaires en btn--lg (46 px) min. Yao bras tendu, terrain humide, gants : pas de cible sous 44 px.

✓ Labels complets

"Enregistrer la saillie", "Valider la mise-bas". Jamais "OK", "Confirmer", "Submit". Le verbe + l'objet métier.

✓ Icône à gauche

Toujours avant le texte, gap 8 px, stroke 2 px, taille proportionnelle (13/14/16/18 selon SM/MD/LG/XL).

✓ Loading non-cliquable

Spinner remplace le texte (transparent), pointer-events:none, on garde la même largeur pour pas faire sauter le layout.

✗ Pas de gradient

Aucun linear-gradient sur fond bouton. Anti-AI feel. Couleur unie, ombre 0 6px 20px max sur le FAB.

✗ Pas de border-radius full

border-radius:50% réservé au FAB. Pas de pill 9999px sur les boutons standards — radius 8/10/12/14 selon taille.

✗ Pas de uppercase forcé

Casse normale. L'uppercase est réservé aux titres display + labels mono. Un bouton uppercase a l'air d'un placard.

✗ Pas de danger en primary

Une suppression part d'un bouton secondary/ghost rouge ou d'un swipe. Le confirm dialog peut promouvoir le danger en primary, jamais avant.

✗ Pas de "Annuler" seul

Un secondary "Annuler" n'apparaît que pair avec un primary. Si annuler est l'unique action → utiliser sheet-close (×).
