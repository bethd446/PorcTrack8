Notes design

3 décisions · 2 hésitations · effort dev

3 décisions design fortes

01 · Pharmacie

Sub-tabs par catégorie, pas chips filtres

Chips = filtre temporaire et perdable au scroll. Sub-tabs = engagement de l'utilisateur sur une catégorie (Vaccins / Antibio / Vermifuges / Autres). Pour la pharmacie où Yao cherche un produit précis, le sub-tab persiste, garde le compteur visible, et reste accessible à 1 tap. Cohérent avec le pattern Élevage `Truies / Verrats / Porcelets / Bandes / Loges`.

02 · Plan d'alimentation

Table verticale · pas grid horizontal

Grid horizontal phase × formule × ration aurait débordé sur 390px et caché la dernière colonne. Table verticale rend Phase / Formule / Ration / État sur 4 colonnes courtes, lisibles d'un coup. La projection 30 j sous la table en `dl.kv` (matières × besoin) consolide en mono — exactement le DNA cahier d'éleveur.

03 · Repro En cours

Mini cycle rail intégré dans la card

Hésité avec expansion au clic — friction supplémentaire pour 3 cycles seulement. Choix : rail 5 nœuds compact (hauteur 18px) directement dans la card, avec labels mono J/MB/SEV en dessous. Yao voit la phase exacte sans tap. Réutilise exactement le pattern `cycle__rail` de la fiche truie, version mini.

2 zones d'hésitation

Hésité · Aliments

Stockbar visuelle ou pill seul ?

Pill `OK / Bas / Rupture` aurait suffi en pure densité métier. J'ai gardé un mini stockbar 48×6px sous le sub-titre — il donne un repère visuel instantané (« 24% restant ») sans casser la grille. Risque : si tu trouves trop décoratif, suffit de retirer la classe `.stockbar` — tout le pattern survit.

Hésité · Historique

Cards stackées vs DataTable seule

Tenté DataTable seule (gain de place, mode avancé natif) → trop sec, perd l'identité bande (Q4 2025, eyebrow année). Choisi : cards stackées en mode standard + DataTable en mode avancé sur la même page. Yao voit les bandes comme objets vivants ; Aïssa exporte en CSV. Risque écarté : forcer Aïssa à scroller des cards quand elle veut juste un tableau triable.
