## Détail par point

### P0 — Bloquants (4)

#### 1. autoFocus absent dans 36/37 forms (P0)
**Constat** : seul `QuickPeseeForm` a `autoFocus` sur son 1er input. Les 36 autres forms forcent l'éleveur à 1 tap supplémentaire pour activer le clavier.
**Référence** : `grep -L 'autoFocus' src/components/forms/Quick*.tsx` → 36 résultats.
**Recommandation** : ajouter `autoFocus` sur le 1er `IonInput` ou `<input>` de chaque form Quick*.

#### 2. Virgule décimale FR non gérée dans 9/36 forms (P0, F12)
**Constat** : forms acceptent type=number mais ne convertissent pas `10,5` (FR) → `10.5`. L'éleveur tape virgule, la valeur reste vide ou échoue silencieusement.
**Référence** : `QuickAddTruieForm.tsx`, `QuickAddPorceletForm.tsx`, `QuickAddBoarForm.tsx`, `QuickPeseeForm.tsx`, `QuickAddSaillieForm.tsx`, `QuickRefillForm.tsx`, `QuickVenteForm.tsx`, `QuickAddBatchSourceForm.tsx`, `QuickAddBandeFromLogeForm.tsx` (parseFloat sans `.replace(',', '.')`).
**Recommandation** : créer `src/lib/parseDecimal.ts` exportant `parseDecimalFR(s) => s.replace(',', '.') |> parseFloat`. Codemod sur les 9 forms.

#### 3. z-index FAB sous nav bar (P0)
**Constat** : `SaisirFAB.tsx` utilise `z-[55]` mais la bottom nav `IonTabBar` a `z-index: 1000` côté Ionic — le FAB peut être masqué par la nav.
**Référence** : `src/components/SaisirFAB.tsx:42`.
**Recommandation** : passer le FAB à `z-[1010]`.

#### 4. Données potentiellement perdues (P0, F5)
**Constat** : si network down pendant submit Quick* form, certains forms n'ont pas fallback offline queue. Risque de perte saisie terrain.
**Référence** : `QuickAddBandeFromLogeForm.tsx` ne passe pas par `enqueueUpdateRow` en mode offline. Comparer avec `QuickEditTruieForm` qui le fait.
**Recommandation** : audit complet des 36 forms, ajouter fallback `enqueueUpdateRow` dans le catch network.

### P1 — Importants (11)

#### 5. Typo incohérente : 1105 `font-mono` vs 52 `ft-*` (P1)
**Constat** : Tailwind utilities `font-mono`/`text-[12px]` utilisés massivement à la place des classes design system `.ft-heading` / `.ft-code` / `.ft-values`. Risque incohérence si la police DMMono est swapée.
**Référence** : `grep -rn "font-mono" src --include="*.tsx" | wc -l` → 1105.
**Recommandation** : migration progressive vers les classes design system, en commençant par les composants partagés (AlertCard, AnimalListItem).

#### 6. Styles inline `style={{}}` : 1196 occurrences (P1)
**Constat** : nombreux composants utilisent `style={{ padding: '10px 16px' }}` ou `style={{ minHeight: 44 }}` au lieu d'utilities Tailwind.
**Référence** : `grep -rn "style={{" src --include="*.tsx" | wc -l` → 1196.
**Recommandation** : codemod `style={{ minHeight: N }}` → `min-h-[Npx]`. Priorité AlertCard, EmptyState, AgritechHeader.

#### 7. fontSize inline : 455 occurrences (P1)
**Constat** : `style={{ fontSize: '12px' }}` partout au lieu de classes Tailwind ou design tokens.
**Référence** : `grep -rn 'fontSize' src --include="*.tsx" | wc -l` → 455.
**Recommandation** : remplacer par `text-[12px]` ou tokens (`text-caption`, `text-mono-micro`).

#### 8. Contraste limite WCAG (P1, F3)
**Constat** : 90 occurrences de `text-text-2` (muted) avec `text-[10px]` — texte 10px en couleur muted = contraste limite plein soleil.
**Référence** : `grep -rn "text-\[10px\].*text-text-2\|text-text-2.*text-\[10px\]" src --include="*.tsx" | wc -l` → 90.
**Recommandation** : passer ces textes à `text-[11px]` minimum + couleur `text-text-1`.

#### 9. CLAUDE.md obsolète (P1)
**Constat** : la doc projet mentionne tokens `emerald-premium` (n'existe plus dans `tailwind.config.js`), `PremiumHeader` (composant supprimé V8), 4 fonts (en réalité 3 fonts utilisées : BigShoulders, InstrumentSans, DMMono).
**Référence** : `CLAUDE.md` lignes 60-95.
**Recommandation** : refresh complet CLAUDE.md avec les vrais tokens et composants actuels.

#### 10. Empty states absents (P1)
**Constat** : seules `BandesView` et `TruiesView` ont un empty state propre. Les autres listes vides affichent rien (page blanche).
**Référence** : `grep -rn "EmptyState\|empty-state\|aucun" src/features --include="*.tsx" -l` → 8 fichiers seulement.
**Recommandation** : créer un composant `<EmptyState icon label hint />` (déjà partiellement existant) et l'utiliser sur toutes les listes.

#### 11. Toasts erreur manquants (P1, F2)
**Constat** : 14 forms catchent les exceptions silencieusement avec `console.error` sans toast utilisateur.
**Référence** : `grep -B1 -A2 "console.error" src/components/forms/*.tsx | grep -c "showToast\|setToast"` → ratio 14/36.
**Recommandation** : tout catch DB doit afficher un toast tone='error'.

#### 12. Tap targets < 44px (P1, F1)
**Constat** : 23 boutons avec `h-[36]`, `h-[40]`, `min-h-[40px]` < 44px critique persona.
**Référence** : `grep -rn "h-\[3[0-9]\|h-\[4[0-3]\]\|min-h-\[3" src --include="*.tsx" | wc -l` → 23.
**Recommandation** : audit + remplacement par `min-h-[44px]` ou `h-[44px]`.

#### 13. Hiérarchie boutons : 5 primary par écran (P1)
**Constat** : sur certaines pages (BandeDetailView), 4-5 boutons utilisent `bg-accent` créant ambiguïté hiérarchique. 1 primary recommandé.
**Référence** : `BandeDetailView.tsx` lignes 280-310.
**Recommandation** : un seul bouton `bg-accent`, les autres en `border` ou `ghost`.

#### 14. aria-label manquant sur boutons icône (P1)
**Constat** : 18 boutons icône-seule sans `aria-label` (pas accessibles screen readers).
**Référence** : `grep -B1 "lucide-react" src --include="*.tsx" -rn | grep -B1 -A2 "<button" | wc -l`.
**Recommandation** : ajouter `aria-label` sur tous les boutons icône.

#### 15. Validation inline incohérente (P1)
**Constat** : certains forms affichent erreurs en haut, d'autres sous chaque champ. Inconsistance qui désoriente.
**Référence** : `QuickAddBandeFromLogeForm.tsx` (haut) vs `QuickAddTruieForm.tsx` (sous champ).
**Recommandation** : standardiser : erreurs sous chaque champ + résumé en haut si multi-erreurs.
