Règles d'usage

5 do · 4 don't · code snippet officiel

✓ Stroke 2 constant

Tous les SVG Lucide en stroke-width:2. Ne jamais alléger en 1.5 — perd la lisibilité sous 16 px à bras tendu sur Android budget.

✓ Couleur = currentColor

Les icônes héritent color du parent. Permet de basculer success/warning/danger sans dupliquer le SVG.

✓ Carré-fond pour actions clés

Bouton ou source : icône dans un carré radius 10–14 px, fond warm + couleur primary. Donne du poids visuel.

✓ Pair sémantique

Toujours préférer la paire success/danger Lucide existante (check-circle ↔ alert-octagon) plutôt que d'inventer.

✓ Loader-2 = spin 1 s

Une seule animation pour tous les loadings : animation:spin 1s linear infinite. Pas de variantes par contexte.

✗ Pas d'emoji

🐷🍼📊 interdits. Anti-AI feel. Yao perçoit l'emoji comme jouet — pas un outil de travail. Lucide stroke uniquement.

✗ Pas d'icône d'entité

Truies, verrats, porcelets, bandes, loges = avatars colorés avec code court mono (T31, V03, M01). Pas de pictogramme animal.

✗ Pas de fill

Lucide est outline-only. Si besoin de "rempli" sémantique (état actif), passe par fond plein du conteneur, pas par fill du SVG.

✗ Pas de gradient

Aucun linearGradient dans les SVG. Couleur unique héritée. Anti-AI feel.

Snippet officiel

Inclusion via Lucide UMD · 1 ligne head + auto-render

<!-- 1. dans le <head> -->
<script src="https://unpkg.com/lucide@latest/dist/umd/lucide.min.js"></script>

<!-- 2. dans le HTML -->
<i data-lucide="bell"></i>
<i data-lucide="check-circle" style="width:18px;height:18px"></i>

<!-- 3. en fin de <body> -->
<script> if(window.lucide) lucide.createIcons(); </script>
